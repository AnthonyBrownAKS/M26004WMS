# WMS系统流程说明
## 入库
### 绑定容器与材料(material_container)
1. 页面用户扫描, 分两次最终解析QR获取 `Scan(materialCode, quantity, containerId, customerCode)`
2. 扫描页面布局参考原系统, 具体如下:
 ![ScanWeb](docs/pics/IMG_20260421_110149.jpg)  

3. 在执行入库操作时, 默认定义当前库区(locationAreaId)为 'C';
4. 假如库存(inventory)已经存在当前容器(containerId), 清空容器属性:
 - update (location_area_id = C, row_no = 1, column_no = 1, container_id) into inventory  


### 页面点击确定直接绑定(待定) 
1. `insert (material_code, container_id, quantity) into material_container`
2. batch 同 creation_time, 例如(2026428)  

### 接口1: 分配库位给WCS层(`location_area_id, row_no, column_no`) 
1. 带权分配(优先选择) (待定)
2. 使用Result封装返回`location_area_id, row, column`
3. 异常处理：没有库位等...status:-1为错误

### 接口2: WCS层返回数据插入库存(`location_area_id, row_no, column_no, container_id`)
 - 从WCS层获取 insert (location_area_id, row_no, column_no, container_id) into inventory
 delete (location_area_id = C, row_no = 1, column_no = 1, container_id) from inventory
 - 异常处理：货位数据重复...使用Result封装返回success/error

# 出库
## 用户选择出库物料
 - 页面流程参考![OutWeb](docs\pics\IMG_20260421_105332.jpg)
 - 排除检入容器与质量状态
 - 可选material_id 或 batch进行筛选, 显示容器ID与当前容器所存material_id与物料对应供应商customer_code
 - 最终提供(material_id, batch, container_id)



## 记录出库任务状态
 - 变更字段`task_status, source_location_id`
 - 判断: 如果`source_location_id`重复且出库任务状态为`CREATED`, 则发送错误报警
 - 提供两个接口:
   1. 获取当前出库任务: 发送时间最早的`CREATED`状态出库任务(`id, source_loaction_id, target_location_id(EMPTY), status`)
   2. 出库完成: 传入任务`id与status`, 检查任务列表是否存在符合数据, 更新`status`为`FINISHED`, 删除inventory对应数据
 - WCS执行

- 解除绑定`material_container`
