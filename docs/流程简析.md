# WMS系统流程说明
## 入库
### 绑定容器与材料(material_container)
1. 页面用户扫描, 分两次最终解析QR获取 `Scan(materialCode, quantity, containerId, customerCode)`
2. 扫描页面布局参考原系统, 具体如下:
 ![ScanWeb](docs/pics/IMG_20260421_110149.jpg)  

3. 在执行入库操作时, 默认定义当前库区(locationAreaId)为 'C';
4. 假如库存(inventory)已经存在当前容器(containerId), 清空容器属性:
 - update (location_area_id = C, row_no = 1, column_no = 1, container_id) into inventory  


### 页面点击确定直接绑定
1. 对应方法入口:
2. 绑定容器与物料`insert (material_code, container_id, quantity) into material_container`
3. batch 同 creation_time, 例如(2026428)  

### 接口1: 分配库位给WCS层(`location_area_id, row_no, column_no`) 
1. 带权分配(优先选择) (待定)
2. 使用Result封装返回`location_area_id, row, column`
3. 异常处理：没有库位等...status: 500 为错误

### 接口2: WCS层返回数据插入库存(`location_area_id, row_no, column_no, container_id`)
 - 从WCS层获取 insert (location_area_id, row_no, column_no, container_id) into inventory
 delete (location_area_id = C, row_no = 1, column_no = 1, container_id) from inventory
 - 异常处理：货位数据重复...使用Result封装返回success/error

# 出库
## 用户选择出库物料
 - 页面流程参考![OutWeb](docs\pics\IMG_20260421_105332.jpg)
 - 排除检入容器与质量状态
 - 可选material_id 或 batch进行筛选, 显示容器ID与当前容器所存material_id与物料对应供应商customer_code
 - 最终提供(material_code, batch, container_id)

## 记录出库任务状态
 - 传入字段`task_status, source_location_id`
 - 判断: 如果`source_location_id`重复且出库任务状态为`CREATED`, 则发送错误报警
 - 提供两个接口:
   1. 获取当前出库任务: 发送时间最早的`CREATED`状态出库任务(`id, source_loaction_id, target_location_id(null), status`)
   2. 出库完成: 传入任务`id与status`, 检查任务列表是否存在符合数据, 更新`status`为`FINISHED`, 删除inventory对应数据

## 用户扫码指定出库物料
- 对应方法入口在于TaskController的`createOutbound`
- 用户扫码提供`Scan(materialCode, quantity, containerId, customerCode)`
- 解除绑定`material_container`, 表中删除字段

# 前端功能
1. 入库页面: 入库记录增删改查, 扫一扫调整光标到初始输入框, 提交出库
2. 出库页面: 出库记录增删改查, 扫一扫调整光标到初始输入框, 提交出库
3. 物料管理页面: 增加/删除/修改/依据物料编码查询物料, 导入/导出物料EXCEL表
4. 客商管理页面: 增加/删除/修改/依据客商编码查询物料, 导入/导出客商EXCEL表
5. 库存管理页面: 依据容器ID, 库区查找库存, 新增/修改库存, 点击下拉展示容器存储物料
6. 日志页面: 展示操作与接口调用情况, 可依据时间进行筛选
7. 系统管理页面: 展示接口调用折线图, 查看日志占用内存并清空



# 项目结构介绍
## 启动方式
- 后端(backend): Maven依赖安装后, 启动BackendApplication, 端口8080
- 前端(frontend): Vue依赖安装完后, 使用Node工具 npm dev run启动package.json, 端口5173
- 数据库(sql): 文件夹中建表语句执行
上述步骤做完后在浏览器访问 localhost:5173 进入系统

## Backend结构介绍
1. common:通用工具类, Result为返回格式统一定义
2. config:已废弃, 线程池配置类
3. controller: 前端交互类, 接收与返回前端数据
    - TaskController: 管理出入库任务请求, WCS层接口交互
    - OutboundController: 出库记录的基本操作
    - InboundController: 入库记录的基本操作
    - CustomerSupplierController: 客商表的基本操作
    - ContainerController: 容器表的基本操作, 前端并没有使用
    - InventoryController: 库存表的基本操作
    - LocationController: 库位表的基本操作
    - MaterialController: 物料表的基本操作
    - UserController: 用户表的基本操作, 前端并没有使用
    - LogsController: 操作/接口日志表的基本操作
4. dto:数据流定义
    - __ExcelDTO表示对应数据表Excel导入导出的定义
    - __PageDTO表示对应数据表分页查询的定义
    - __QueryDTO表示返回类型的数据, 已废弃
5. entity:数据表对应的JavaBean类
6. mapper:Mybatis-plus数据库交互字段
7. queue:任务队列,已废弃
8. service:接口与实现类
    - __service:对应数据表的方法接口
    - __serviceImpl:对应对应数据表接口的实现类
- 程序重点在于TaskServiceImpl,出入库操作核心都在其中
- Excel导入导出在于对应数据表的Impl中


## frontend结构介绍
src中存放核心前端代码
1. api:配置后端接口
2. layout:页面基底元素,所有页面的通用边框
3. router:页面跳转设置
4. views:功能页面
    - ApiLog:接口日志
    - Customer:客商管理
    - Inbound:入库管理
    - Outbound:出库管理
    - System:系统管理
    - Inventory:库存管理
    - Material:物料管理
    - OperationLog:操作管理
    - TaskQuery:任务管理
5. App:Vue的主元素
6. vite:代理跨域配置
